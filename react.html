Why use react?

Before we start jumping into our IDE's and getting in the zone, let's discuss why we should even use React in the first place:

Simple: React is easy and flexible.

React is designed from the ground up to be easily maintained, robust, and integratable. You can use it on websites, web applications, and even in desktop applications.

React uses "Components" which are chunks of code that are reusable and independant. This makes it incredibly easy to integrate React and eventually convert existing code into React. This is actually exactly what Facebook / Meta did. Once they created React, they began to rewrite the entirety of Facebook and Instagram to use it, and as far as popular websites with good UI go, Facebook and moreso Instagram definitely come to mind.

Step 1: Add React to your HTML

I prefer to use Visual Studio Code for webdev coding, so this tutorial will be specific for that. Regardless of what your IDE of choice is, you should be able to follow along.

To get React running on our site, we need to add it to the JavaScript sources.

Make sure to place these source tags just above your closing </body> tag.


                        
                            
                                <script src="https://unpkg.com/react@18/umd/react.development.js" crossorigin></script>
                                <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js" crossorigin></script>
                                <script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>
                                </body>

This step is self explanatory: we're just telling the browser to load the React library, the same way we tell it to load our local script files. The reason we want this script tag at the bottom is so that the website's content get loaded before the browser starts to grab and parse all the libraries the site is asking for. By and large, this is a quality-of-life feature. We don't want our user to be waiting a year before the website loads because it's grabbing libaries.

If you prefer to keep your script tags at the top of the page (like me) an alternate way of making sure the page loads fast while still having your script tags at the top is to use defer in your opening script tag. 

                            
                                
                                    <script defer src="https://unpkg.com/react@18/umd/react.development.js" crossorigin></script>
                                    <script defer src="https://unpkg.com/react-dom@18/umd/react-dom.development.js" crossorigin></script>
                                    <script defer src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>

For this tutorial, we're going to keep it simple and just place our source tags near the bottom.

You might notice that the last line there is importing something called Babel. We're going to use Babel, because some of our code is going to be JSX, which is essentially React's way of putting HTML inside of JavaScript. It'll come in handy as Babel is a JavaScript transcompiler that essentially can translate the latest JavaScript into whatever potentially older JavaScript the browser is compatible with. It's a compatibility guarantee, and is what's going to let us get away with some of the weird hybrid HTML/JS we're coding.

Step 2: Create and Add a Component

The first part of this step is figuring out where on the page we want to put a React Component.

In this case, lets put it near the top of the page where it's easily viewable.

Create a new Div and give it an ID.

                            
                                
                                <div id="PutReactHere"></div>

Next, we need to start writing some React.js. In your IDE of choice, create a new JavaScript file and name it whatever you want.

I'm going to name mine LearnReact.js.

I want to have a React Component that is a button, and when I click the button I want it to display "Powered by React.js".

So, let's open up our new LearnReact.js, and punch in the following:

                            
                                
                                class LearnReact extends React.Component {
                                    constructor(props){
                                        super(props);
                                        this.state = {
                                            clickMe: false
                                        }
                                    }
                                
                                    render(){
                                
                                        if(this.state.clickMe){
                                            return (
                                                <h2>Powered by React.js</h2>
                                            )
                                        }
                                
                                        return (
                                            <button onClick={()=> this.setState({clickMe: true})}>Click Me!</button>
                                
                                        )
                                    }
                                }

Step 3: Understand

Okay, let's break down what we have so far, step by step, and make sure we know what's going on.

We have our imports set up. React's library is loaded, ReactDOM is loaded, and Babel is loaded. We already know what Babel is for, so let's briefly talk about the others.

Well, the React library is... React. Without this import, all the component code in Step 2 simply won't work.

ReactDOM is a package that handles DOM specific methods, namely in our case, render().

A quick review of DOM: DOM stands for Document Object Model. This is the data representation of everything on the website. By loading ReactDOM, we can manipulate the DOM (and by extension, the site itself) by using React-specific methods.

Alright, we're up to speed. Let's break down the React code itself.

                            
                                
                                class LearnReact extends React.Component {
                                    constructor(props){
                                        super(props);
                                        this.state = {
                                            clickMe: false
                                        }
                                    }

This first chunk is the Component itself. We have created a new class which inherits React.Component, and we gave it a constructor which has "props". Props is short for properties. It's really straightforward - Our class has properties. Kind of like HTML attributes, or function arguments in vanilla JS.

We need the constructor so that the properties are initiated. By including super(props), we are executing the parent component's (React.Component) constructor function. This makes sure our component, LearnReact can do anything React.Component can.

React properties are stored in an object called state.

We have only 1 property, which is the clickMe property. It's a boolean, and we're initializing it as false.

                            
                                
                                    render(){
                                
                                        if(this.state.clickMe){
                                            return (
                                                <h2>Powered by React.js</h2>
                                            )
                                        }

The render() section is for everything we want React to output. This chunk of code is checking if the clickMe property is set to true - and if it is, the ReactDOM should render out a new chunk of HTML.

This code needs to come before the next chunk or it will be unreachable, because the next chunk of code gives the render method a return. 

                            
                                
                                        return (
                                            <button onClick={()=> this.setState({clickMe: true})}>Click Me!</button>
                                
                                        )
                                    }
                                }

So, after the Component loads, this return creates a button, which via the onClick event handler, is able to change the clickMe property from false to true.

When React Components' state changes, the component reloads - and since the state is changed to true, when the Component reloads it will create our "Powered by React" header instead of the button.

Step 4: Tell React and Babel To Do Their Thing

We're doing pretty good so far! Our React JSX is all in place. Our button works, and we have a good understanding of how Components work. If we wanted to, we could reuse the Component we have for all sorts of buttons on our website.

Wait. Our button works? No. It doesn't. We still need to import our LearnReact.js file. And what about all the JSX we coded? We need Babel to kick in to translate all of that to the browser.

Let's switch over to our index.html and finish up.

Right below our previous imports, let's add the following:

                            
                                
                                <script type="text/babel" src="LearnReact.js"></script>
                                <script type="text/babel">
                                    ReactDOM.render(<LearnReact/>, document.getElementById('PutReactHere'));
                                </script>

Now we're done - by adding the type tag to these imports, we're making sure the HTML page is grabbing all this React JSX code after Babel has deciphered it for us.

We've also added one last tiny script which calls ReactDOM.render to put its returns inside the Div we created earlier.
